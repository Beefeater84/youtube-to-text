# submitTranscriptJob Server Action

- Получает на вход videoId и languages
- Проверяет авторизацию пользователя
- Проверяет, есть ли уже запись с таким videoId и languages
- Сохраняет запись в базу данных


**Use Case 1:**

- Пользователь не авторизован

Ожидаемое поведение:

- Возвращается ошибка "You must be signed in."

**Use Case 2:**

- Видео с videoId нет в базе данных
- Languages: ["en", "ru"]

Ожидаемое поведение:

- Записываем запись в базу данных с языком en
- Записываем запись в базу данных с языком ru

**Use Case 3:**

- Видео с videoId есть в базе данных
- Languages: ["en", "ru"]

Ожидаемое поведение:

- Проверяем все языки, которые есть в базе данных для этого videoId
- Проверяем status всех языков, если status не "done" и не "processing", и не "queued", то обновляем status на "pending", чтобы worker мог снова взять это видео в работу
- Если язык уже есть в базе данных, то не сохраняем запись
- Если язык нет в базе данных, то сохраняем запись
- Возвращается success: true  

**Use Case 3:**

- Видео с videoId есть в базе данных
- Languages: ["en", "ru"]

Ожидаемое поведение:

- Если все языки уже есть в базе данных, то возвращается success: true

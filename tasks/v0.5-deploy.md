# Задача v0.5: Деплой на Hetzner VPS

Этот план описывает последовательность действий для развертывания Next.js приложения и Python-воркера с использованием конфигурации из папки `deploy`.

## 1. Подготовка инфраструктуры (Сервер и Домен)
- [ ] Зарегистрироваться/авторизоваться в Hetzner Cloud и создать новый сервер (Ubuntu 22.04 LTS или 24.04 LTS).
- [ ] Добавить свой публичный SSH-ключ на сервер при создании.
- [ ] Приобрести доменное имя и создать A-запись, указывающую на публичный IP сервера Hetzner.

## 2. Первоначальная настройка VPS
- [ ] Подключиться к серверу по SSH.
- [ ] Установить Docker Engine и Docker Compose plugin.
- [ ] Создать директорию для деплоя на сервере (`mkdir -p /opt/youtube-to-text`).
- [ ] *Если репозиторий проекта приватный:* залогиниться в GitHub Container Registry (GHCR) на сервере с помощью access-токена.

## 3. Сбор и подготовка переменных окружения (.env)
- [ ] Собрать все необходимые ключи от Supabase (Cloud): `SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`.
- [ ] Скопировать локальный шаблон `deploy/.env.example` в отдельный текстовый документ и заполнить актуальными значениями API ключей.

## 4. Настройка GitHub Actions (CI/CD)
Перейти в настройки репозитория на GitHub -> Settings -> Secrets and variables -> Actions и добавить следующие секреты:
- [ ] `VPS_HOST` — IP-адрес сервера Hetzner.
- [ ] `VPS_PORT` — SSH порт (по умолчанию 22).
- [ ] `VPS_USER` — пользователь сервера (например, root).
- [ ] `VPS_SSH_KEY` — приватный SSH ключ (публичный должен быть в `~/.ssh/authorized_keys` на сервере).
- [ ] `VPS_DEPLOY_PATH` — абсолютный путь к папке проекта на сервере (`/opt/youtube-to-text`).
- [ ] `DEPLOY_ENV_FILE` — полное содержимое подготовленного `.env` файла.

Также необходимо: убедиться в настройках `Packages`, что у GitHub Actions есть права на запись образов в GHCR `write`.

### Настройка домена (Namecheap -> Hetzner)
Чтобы домен вел на твой сервер:
1. Зайди в **Namecheap** -> **Domain List** -> **Manage** -> **Advanced DNS**.
2. Добавь (или измени) **A Record**:
   - Host: `@`
   - Value: `178.104.134.102` (IP твоего сервера Hetzner)
3. Добавь **CNAME Record** (по желанию):
   - Host: `www`
   - Value: `твойдомен.com`
4. Подожди от 30 минут до нескольких часов, пока обновятся DNS.

**После прописки DNS:**
Обнови переменную `DOMAIN_NAME` в своем секрете `DEPLOY_ENV_FILE` (например, `DOMAIN_NAME=yourdomain.com`). Caddy сам увидит домен и автоматически выпишет бесплатный SSL-сертификат от Let's Encrypt.

## 6. Проверка работоспособности
- [ ] Зайти на настроенный домен (https) и убедиться, что сайт доступен (Caddy автоматически занимается SSL).
- [ ] Подключиться на сервер по SSH и проверить логи: `docker compose -f /opt/youtube-to-text/deploy/docker-compose.prod.yml logs worker -f`.
